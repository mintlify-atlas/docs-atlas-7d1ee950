## import_images

Import images into an existing COLMAP database by reading their EXIF metadata and registering camera/image entries — without running SIFT feature extraction.

### Function signature

```python
pycolmap.import_images(
    database_path,
    image_path,
    camera_mode = pycolmap.CameraMode.AUTO,
    image_list  = [],
    options     = pycolmap.ImageReaderOptions(),
)
```

### Parameters

<ParamField path="database_path" type="str or Path" required>
  Path to an existing COLMAP SQLite database. The database must already exist (use `extract_features` to create a new one with feature extraction, or create the database separately first).
</ParamField>

<ParamField path="image_path" type="str or Path" required>
  Root directory containing the images.
</ParamField>

<ParamField path="camera_mode" type="pycolmap.CameraMode" default="CameraMode.AUTO">
  Controls how cameras are assigned to images.

  - `AUTO` — share a camera across images with identical EXIF intrinsics.
  - `SINGLE` — all images share one camera.
  - `PER_FOLDER` — images in the same folder share a camera.
  - `PER_IMAGE` — each image gets its own camera.
</ParamField>

<ParamField path="image_list" type="list[str]" default="[]">
  Optional list of image filenames (relative to `image_path`) to import. When empty, all images in `image_path` are imported.
</ParamField>

<ParamField path="options" type="pycolmap.ImageReaderOptions">
  Image reader configuration including `camera_model`, `camera_params`, `mask_path`, and `default_focal_length_factor`.
</ParamField>

### Return value

<ResponseField name="return" type="None">
  Returns `None`. Camera and image entries are written directly to `database_path`.
</ResponseField>

## infer_camera_from_image

Read an image file and infer a COLMAP `Camera` from its EXIF metadata. No database is involved.

### Function signature

```python
pycolmap.infer_camera_from_image(
    image_path,
    options = pycolmap.ImageReaderOptions(),
)
```

### Parameters

<ParamField path="image_path" type="str or Path" required>
  Path to a single image file (JPEG, PNG, TIFF, etc.).
</ParamField>

<ParamField path="options" type="pycolmap.ImageReaderOptions">
  Reader options. The `camera_model` field controls which model is used (default: `SIMPLE_RADIAL`). The `default_focal_length_factor` is used when EXIF focal length is absent.
</ParamField>

### Return value

<ResponseField name="return" type="pycolmap.Camera">
  A `pycolmap.Camera` object initialized from the image's EXIF data. The `has_prior_focal_length` property is `True` when focal length was read from EXIF, `False` when it was estimated from the image dimensions.
</ResponseField>

## Examples

<CodeGroup>

```python import_images basic
import pycolmap
from pathlib import Path

database_path = Path("database.db")
image_dir     = Path("images/")

# Import images without feature extraction
# (database must already exist)
pycolmap.import_images(database_path, image_dir)
```

```python import_images single camera
import pycolmap
from pathlib import Path

pycolmap.import_images(
    database_path=Path("database.db"),
    image_path=Path("images/"),
    camera_mode=pycolmap.CameraMode.SINGLE,
    options=pycolmap.ImageReaderOptions(camera_model="OPENCV"),
)
```

```python infer_camera_from_image
import pycolmap

camera = pycolmap.infer_camera_from_image("images/frame001.jpg")
print(camera)
print("Has prior focal length:", camera.has_prior_focal_length)
print("Model:", camera.model)
print("Params:", camera.params)
```

</CodeGroup>

<Tip>
  Use `infer_camera_from_image` to get a quick camera estimate for a known image, for example to create a `pycolmap.Camera` object before calling `absolute_pose_estimation`.
</Tip>
